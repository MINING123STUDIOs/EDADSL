\chapter{Introduction}
\section{Abstract}
EDADSL (Electronic Design Automation Domain Specific Language) is a Domain-specific Language (DSL) specialized for programatic Elecectronic Design Automation (EDA) and Hardware-Synthesis. 
It is based on an Electrical System and a constraint driven optimisation Engine\\
This Document is a full Outline and as of now the authoritative Specification of the EDADSL Language as well as the Systems underlying it.

\chapter{Lexical Structure}
\section{Character Set}
EDADSL supports the Full Unicode Character-set.
\section{Whitespace}
Whitespace is defined using the following Regular Expression:
\begin{verbatim}
Whitespace                    = [ \t\n\r]+
\end{verbatim}

\section{Comments}
Comments are defined using the following Regular Expressions:
\begin{verbatim}
Line_Comments                 = "#"[^\n]*
Multiline_Comments            = "#{" [^"}#"]* "}#"
\end{verbatim}

\section{Identifiers}
Identifiers are defined using the following Regular Expressions:
\begin{verbatim}
Variable_Identifiers          =     \$[_a-zA-Z0-9]+
Built_In_Function_Identifiers =    f\.[_a-zA-Z0-9]+
User_Function_Identifiers     = f\.u\.[_a-zA-Z0-9]+
Built_In_Constant_Identifiers =    c\.[_a-zA-Z0-9]+
User_Constant_Identifiers     = c\.u\.[_a-zA-Z0-9]+
\end{verbatim}

\section{Keywords}
The language uses the following Keywords defined using the following Regular Expressions:
\begin{verbatim}
Generic_Keywords:    start halt if for gotomarker gotojump writetolog mkpcb 
                     mkschematic export echo return
Physical_Keywords:   phys\.[a-z]+
Electrical_Keywords: elec\.[a-z]+
System_Keywords:     syst\.[a-z]+
\end{verbatim}

\section{Literals}
For the Lexical Structure of Literals see:  [ NUMBER OF SECTION Types/Type Literals ] Type Literals.

\section{Operators}
For the Lexical Structure of built in Operators see:  [ NUMBER OF SECTION Operators/Built-in Operators ] Built-in Operators. \\
User-defined Operators are defined using the following Regular Expressions:
\begin{verbatim}
User_1_Arg_Operators     = o\.u\.[_a-zA-Z0-9]+
User_Binary_Operators    = o\.u\.[_a-zA-Z0-9]+
\end{verbatim}

\section{Delimiters}
EDADSL uses the following Delimiters:
\begin{verbatim}
"" '' () [] {} . , : |
\end{verbatim}

\section{Tokenization Rules}
The Tokenization follows the following Rules: \\
Longer Matches takes Priority over shorter Matches. \\
Keywords take Priority over Identifiers. \\
Whitespace and Comments are discarded.
